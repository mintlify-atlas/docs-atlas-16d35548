---
title: "esp-hal"
description: "Bare-metal (no_std) Hardware Abstraction Layer for Espressif ESP32 microcontrollers, written in Rust."
---

esp-hal provides safe, idiomatic Rust APIs for programming Espressif ESP32 microcontrollers without an operating system. It supports the full ESP32 family — from the original ESP32 to the latest ESP32-C, H, and S series chips — covering both RISC-V and Xtensa architectures.

## Supported chips

esp-hal runs on every chip in the ESP32 family:

| Series | Chips | Architecture |
|--------|-------|-------------|
| ESP32 | ESP32 | Xtensa LX6 |
| ESP32-C | ESP32-C2, C3, C5, C6, C61 | RISC-V |
| ESP32-H | ESP32-H2 | RISC-V |
| ESP32-S | ESP32-S2, S3 | Xtensa LX7 |

## Key features

<CardGroup cols={2}>
  <Card title="Blocking & async APIs" icon="bolt">
    Every peripheral supports both blocking and async operation modes. Use Embassy for fully async embedded applications.
  </Card>
  <Card title="embedded-hal compatible" icon="plug">
    Implements the `embedded-hal` and `embedded-hal-async` traits, making esp-hal compatible with the wider Rust embedded ecosystem.
  </Card>
  <Card title="Type-safe peripherals" icon="shield-check">
    Peripheral singletons returned from `init()` guarantee exclusive access at compile time, preventing resource conflicts.
  </Card>
  <Card title="DMA support" icon="arrow-right-arrow-left">
    High-throughput transfers for SPI, I2C, UART, I2S, and more — freeing the CPU while data moves in the background.
  </Card>
</CardGroup>
